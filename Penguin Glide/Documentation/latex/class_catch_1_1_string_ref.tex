\subsection{Detailed Description}
A non-\/owning string class (similar to the forthcoming std\+::string\+\_\+view) Note that, because a \doxyref{String\+Ref}{p.}{class_catch_1_1_string_ref} may be a substring of another string, it may not be null terminated. \doxyref{c\+\_\+str()}{p.}{class_catch_1_1_string_ref_a1669cb2765e820ca258159676cbd82a5} must return a null terminated string, however, and so the \doxyref{String\+Ref}{p.}{class_catch_1_1_string_ref} will internally take ownership (taking a copy), if necessary. In theory this ownership is not externally visible -\/ but it does mean (substring) String\+Refs should not be shared between threads.
